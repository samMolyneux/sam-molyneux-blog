Markdown to HTML Reference

A living reference for how markdown syntax maps to styled HTML on this site

Headings

Markdown heading levels map directly to HTML heading elements. # Heading is reserved for the page title inside <hgroup> and should only appear once per page.

This is an h3 — ### Subsection

This is an h4 — #### Sub-subsection

Headings use color: #2c3e50 and scale down at the 768px responsive breakpoint.

Paragraphs

Plain text in markdown becomes a <p> element. Each blank line between text creates a new paragraph. Body text is color: #333 with line-height: 1.6.

This is a second paragraph, demonstrating the default spacing between consecutive <p> tags. Browser-default margins are used throughout — no custom paragraph spacing is applied.

Inline Formatting

Markdown **bold** becomes bold via <strong>.

Markdown *italic* becomes italic via <em>.

Markdown `inline code` becomes inline code via <code>. It renders in the browser's default monospace font.

These can be combined: bold and italic or used with code in the same sentence.

Links

Markdown [text](url) becomes a standard <a> element: this is a content link. Content links use default browser styling (blue, underlined).

Navigation and decorative links use the link-unstyled class to remove colour and underline — this is used in the header and for image-wrapping links, not for inline content links.

Unordered Lists

Markdown - item becomes <ul><li>:

  • First item in an unordered list
  • Second item with bold text and a link
  • Third item with inline code

Default browser bullet styling is used. No custom list margins or styles are applied.

Ordered Lists

Markdown 1. item becomes <ol><li>:

  1. First step in a process
  2. Second step with emphasis
  3. Third step referencing some command

Blockquotes

Markdown > quoted text becomes a <blockquote>:

This is a blockquote. It has a 3px solid #999 left border, text colour #555, and 1em left padding.

Blockquotes can contain multiple block-level elements:

A blockquote with a paragraph followed by a list:

  • Item inside a blockquote
  • Another item inside a blockquote

And another paragraph after the list.

Nested blockquotes are not used on this site. Keep blockquotes flat.

Small / Meta Text

Published dates and disclaimers use <small> inside a paragraph. Dates must be wrapped in a <time datetime="YYYY-MM-DD"> element for machine readability:

Published March 20th 2026

This is a disclaimer or aside rendered at a smaller size.

The datetime attribute uses ISO 8601 format (YYYY-MM-DD) while the visible text uses the human-friendly format (Month Dayth Year).

Images

Markdown ![alt text](src) maps to a full <figure> block. Images are never placed inside text sections — they always get their own block.

There are two image classes:

  • feature-image-landscape — 100% width, for wide/landscape images
  • feature-image-portrait — 50% width, for tall/portrait images

Both require the base feature-image class. All images must include width and height attributes (actual pixel dimensions) to prevent layout shift. All images except the first on each page should have loading="lazy".

Thematic Breaks

Markdown --- does not map to an <hr> tag. Instead, start a new <section class="block">. The visual separation between blocks (whitesmoke boxes with gaps between them) serves as the thematic break.

This is a new section block — the gap between this and the previous block is how thematic separation is achieved on this site.

Heading Groups

The <hgroup> element pairs a heading with a subtitle or date range. It is used for post titles (with <h1>) and listing entries (with <h2>). Margins on headings inside <hgroup> are reset to 0.

Example Post Title

January 3rd to 5th 2026

The date range paragraph inside <hgroup> is used for bikepacking posts. Tech posts typically omit it.

Footer content uses <footer class="desc"> rather than <section>. On blog posts this contains the "time machine" link. On listing pages it also includes an RSS link.
